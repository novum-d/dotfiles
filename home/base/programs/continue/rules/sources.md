# Source Policies

- Prefer official documentation whenever available.
- Use community sources only when official documentation is insufficient.
- Prefer sources matching the current SDK, OS, compiler, or tool version.
- Verify publication dates for rapidly changing technologies.
- Avoid outdated Android BLE guidance prior to Android 12 unless explicitly required.
- Prefer modern Kotlin Flow and Coroutine guidance over deprecated patterns.
- Prefer current Nix flakes documentation when applicable.
- Prefer current Android behavior-change documentation for recent API levels.
- Prefer primary documentation over summarized blog content when possible.
